### II. IMMEDIATE MANDATE: EXTEND SINGLE-PASS BRANCHING TO 'ELSE'
Building directly on your successful forward-jump patch stack mechanics (where IF placeholder offsets are backfilled via displacement calculations), implement the complete `if / else` syntax block protocol:

1. Phase 1: Handling 'if' Failures:
   - When processing `if <condition>:`, emit a conditional jump instruction (e.g., `JNE`) with a temporary 4-byte `0x00000000` placeholder operand. Push its buffer location onto `jump_patch_stack`.

2. Phase 2: Encountering the 'else:' Token:
   - When the parser matches `TOK_ELSE` right as the `if` body block scopes close:
     A. Immediately emit an unconditional jump instruction (`JMP` / `0xE9`) with a fresh 4-byte `0x00000000` placeholder operand right at the tail end of the `if` body.
     B. Pop the original `if` failure placeholder address off the `jump_patch_stack`.
     C. Compute the failure relative displacement: `Displacement = Current_Output_Buffer_Pointer - (IF_Placeholder_Address + 4)`. Overwrite those old placeholder bytes so an evaluation failure routes directly to the start of this `else` block.
     D. Push the address of your newly emitted `JMP` placeholder onto the `jump_patch_stack`.

3. Phase 3: Exiting the 'else:' Body:
   - When processing the `TOK_DEDENT` that terminates the `else` block scope:
     A. Pop the `JMP` placeholder address off the stack.
     B. Calculate the exit path displacement: `Exit_Displacement = Current_Output_Buffer_Pointer - (JMP_Placeholder_Address + 4)`.
     C. Patch those 4 placeholder bytes with `Exit_Displacement` so successful `if` runs skip the `else` block entirely.
